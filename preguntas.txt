1. Cuando se genera el archivo obj a partir del archivo “.asm” ¿Qué información nos muestra en consola y qué significa?
    Nos muestra que archivo estamos utilizando para la conversión, los errores que podemos tener en el código (Error messages), los mensajes de advertencia (Warning messages) para mostrar cosas que podrían causar problemas o que es mejor que las arreglemos, el número de veces que va a pasar por el archivo (passes) que por defecto es 1 y la memoria que utilizó para convertir el archivo.
2. ¿En qué punto se crea el archivo ejecutable?
    Se crea luego de ejecutar el TLINK con el archivo .obj
3. ¿Cuál es la interrupción utilizada para imprimir una cadena de caracteres?
    Es la 09H
4. ¿Cuál es la interrupción para imprimir solamente un caracter?
    Es la 02
